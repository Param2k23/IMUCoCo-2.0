Sensor Location Classification - Methods and Results
===================================================

1) Problem
----------
Goal: predict IMU sensor location among 24 body regions from inertial time-series data.

Classes:
- 24 SMPL/IMUCoCo regions (label IDs 0..23)


2) Data Type and Format
-----------------------
Source data:
- Hugging Face parquet shards:
  - train-*.parquet
  - test-*.parquet

Processed training/eval format (.npz):
- X: shape (N, 9, T), float32
- y: shape (N,), int64 labels in [0, 23]
- subject_ids: shape (N,), int64

Meaning of dimensions:
- N = number of windows (samples)
- 9 = channels per window:
  - r6d_0..r6d_5 (6D orientation)
  - ax, ay, az (acceleration)
- T = number of timesteps per window
  - in these runs, T = 300
  - T=300 means 300 samples/frames per window (not 300 seconds)

Subject metadata:
- Parsed from parquet column: subject_id


3) Data Used in Current Runs
----------------------------
Fixed-split run:
- Train: X=(21240, 9, 300), classes=24, subjects=8
- Test:  X=(456, 9, 300), classes=24, subjects=2

LOSO run (merged train+test):
- X=(21696, 9, 300)
- y=(21696,), classes=24
- subject_ids=(21696,), subjects=10 (IDs 1..10)


4) Model and Training Method
----------------------------
Model:
- ResNet1D (model.py)
- Pipeline: Conv stem -> residual blocks -> global average pooling -> dropout -> FC(24)

Training setup (train.py):
- Loss: CrossEntropy
- Optimizer: Adam
- LR scheduler: CosineAnnealingLR
- Early stopping with patience
- Train-only normalization stats (saved and reused at eval)

Evaluation (evaluate.py):
- Per-window accuracy
- Majority-vote accuracy (vote_k=5)
- Confusion matrix
- Symmetry confusion analysis
- Spatial error (meters) for misclassified windows


5) Methods Executed
-------------------
A) Baseline fixed split
- preprocess_vimu.py --mode hf_parquet --hf_split predefined
- train.py --mode fixed_split
- evaluate.py on test split

B) LOSO baseline
- Merge processed train/test into single loso_dataset.npz
- train.py --mode loso (subject-wise folds)

C) LOSO tuning
- Script: run_loso_tune.slurm
- Hyperparameter sweep (first 5 folds):
  - trial_a: epochs=80,  bs=128, lr=1e-3, wd=1e-4, pat=15
  - trial_b: epochs=100, bs=128, lr=1e-3, wd=3e-4, pat=20
  - trial_c: epochs=80,  bs=64,  lr=1e-3, wd=1e-4, pat=15
  - trial_d: epochs=80,  bs=128, lr=5e-4, wd=1e-4, pat=20
  - trial_e: epochs=120, bs=128, lr=7e-4, wd=3e-4, pat=25
- Best trial selected from sweep leaderboard
- Best config rerun on full LOSO (all folds)

7) Main Takeaways
-----------------
- Fixed split baseline: 0.7171 per-window accuracy.
- LOSO baseline improved to 0.7517 mean across 10 subjects.
- LOSO tuning improved further to 0.7817 mean LOSO.
- Best sweep config used stronger regularization (wd=3e-4) and longer training.
- Majority-vote metric is weak in current setup due to very low lock fraction.
